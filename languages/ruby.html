Zastosowanie

Ruby to interpretowany, w pełni obiektowy język programowania stworzony przez Yukihiro Matsumoto. Jako bardzo prosty język jest często stosowany do pisania tak zwanych jednolinijkowców, skryptów administratorskich dla administratorów systemów *unix, przetwarzania tekstu oraz prototypowania nowych aplikacji. Jednak jego największym zastosowaniem jest ogromna ilość frameworków webowych oraz aplikacji napisanych w tym jezyku. Jekyll - generator stron statycznych, napisany właśnie w tym języku pozwala nam na proste tworzenie blogów za pomocą wcześniej napisanych w HTML oraz CSS szablonów, i wygenerowanie gotowych plików, które pozostaje tylko wrzucić na serwer. Używa go Github Pages - serwis Githuba pozwalający na stworzenie własnej prostej strony i osadzienie na domenie *.github.com.

Ruby on Rails jest frameworkiem służącym do tworzenia aplikacji internetowych napisanym w języku Ruby. Zostały zaprojektowane aby programista mógł stworzyć funkcjonującą aplikację w bardzo małej ilości kodu. Rails działa według założeń swojego twórcy, David'a Heinemeier Hansson: "Ruby jest przede wszystkim językiem do pisania pięknego kodu, który czyni programistów szczęśliwymi. Biorąc to pod uwagę, Rails jest próbą połączenia piękna i produktywności Ruby do tworzenia aplikacji webowych.". Ruby on Rails posiada jedną z największych społeczności, która bardzo chętnie dzieli się swoimi dokonaniami i wiedzą, ponadto możemy kupić wiele książek pozwalających na szybką naukę. Ruby posiada wiele zalet, między innymiw przejrzysty sposób współpracuje z różnymi bazami danych, od SQLite przez MySQL i PostgreSQL po MSSQL czy Oracle, nie wymagając przy okazji zmiany kodu przez programistę. Jednak posiada on jedna wadę - wydajność. W większości wypadków strony stworzone w tym frameworku nie należą do demonów szybkości, co jest ceną za wygodę pracy oraz szybkość tworzenia aplikacji.

Poziom skomplikowania

Ruby jest jednym z najprostszych języków. Posiada bardzo intuicyjną składnię, której nauka przychodzi bardzo szybko, nawet początkującym adeptom kodu. Porównajmy wypisanie "Hello World" przy użyciu kilku języków:

          Ruby
          puts "Hello World"

          C++
          #include  using namespace std int main { cout << "Hello, world!" << endl; return 0; }

          Java
          public class HelloWorld { public static void main(String [] args) { System.out.println("Hello World!"); }}

          Assembly
          section .data
             msg     db      "Hello, world!",0xa
             len     equ     $ - msg
           section .text
             global _start
           _start:
             mov     edx,len
             mov     ecx,msg
             mov     ebx,1
             mov     eax,4
             int     0x80
           mov ebx,0
             mov     eax,1
             int     0x80

Od razu widać, który język jest najprostszy. Ruby, jako dynamicznie pisany język posiada wiele możliwości na napisanie tego samego, a na dodatek czytanie go jest praktycznie identyczne z czytaniem instrukcji po angielsku.

Wydajność

Ruby ze względu na swoją dynamiczność jest bardzo cięzki w utrzymaniu podczas pisania większych projektów, co niesie za sobą bardzo trudne debugowanie. Dlatego pisanie dużych aplikacji przy pomocy tego języka wymaga odpowiednio dużego doświadczenia. Dodatkowo, Ruby jest dość wolnym językiem, głównie ze względu na swoją elastyczność - maszyna musi sprawdzić dużo odniesień co znacznie spowalnia działanie programu, w porównaniu z statycznie pisanymi językami. Natomiast JRuby - implementacja Ruby w Javie, mimo, że nie tak szybka jak sama Java pozwala dużo nadrobić jeśli chodzi o prędkość tego języka.

Popularność

W sieci dostępnych jest wiele stron zrzeszających zwolenników Ruby, gdzie doświadczeni programiści chętnie udzielają pomocy i porad nowym użytkownikom, co pozwala na często, szybkie rozwiązanie danego problemu. Ruby posiada ósmą społeczność na StackOverflow, z 175,281 pytaniami. Mimo że nie jest to dużo, społeczność ta jest druga pod względem częstotliwości odpowiedzi na pytania - opisując problem masz bardzo dużą szansę na dostanie przydatnej odpowiedzi. Natomiast na stronie Meetup.com Ruby obecnie posiada 603,759 na całym świecie, z liczbą ponad 910 zarejestrowanych spotkań, zdecydowanie programista szukający grupy osób do nauki znajdzie coś dla siebie.

Na Githubie jest to 3 najczęściej tagowany język. Ogromna ilość różnych projektów, pozwala Ci na użycie ich jako "gemów" w swoim projekcie, jak i różnych narzędzi ułatwiających tworzenie aplikacji.
