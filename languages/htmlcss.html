Czym jest HTML i CSS?

Jeśli chcesz zacząć tworzyć strony internetowe powinieneś zacząć właśnie od tych technologii. HTML (ang. HyperText Markup Language) - jest to język znaczników za pomocą, którego można tworzyć strukturę strony internetowej, a CSS (Cascading Style Sheets - kaskadowe arkusze stylów) nadaje jej wyglądu. Wraz z JavaScript składają się na podstawowy zestaw umiejętności dla frontend developera.

HTML

Znaczniki HTML są budulcem, za pomocą których tworzy się strukturę strony internetowej (wstawianie obrazków, akapitów, nagłówków, dzielenie strony na sekcje itd.) W sekcji nagłówkowej zdefinujemy kodowanie znaków, język, tytuł strony, jej opis itp. W pliku HTML załączymy także arkusze stylów CSS, skrypty Javascript czy PHP. Nadamy identyfikatory i klasy poszczególnych elementów dzięki czemu można uzyskać do nich dostęp w w powyższych technologiach. Ale to są tylko podstawowe możliwości. To jak zaprojektujemy stronę wpływa na jej pozycję w wyszukiwarce oraz dostepność dla osób i urządzeń z ograniczonymi możliwościami, więc nie powinniśmy traktować HTML pobłażliwie.

CSS

CSS służy do stylizowania strony napisanej w HTML. To w nim zdefinujemy kolory, efekty, położenie, sposób wyświetlania a nawet animacje poszczególnych elementów. Został stworzony w celu odseparowania struktury dokumentu od formy jego prezentacji. Separacja ta zwiększa zakres dostępności witryny, zmniejsza zawiłość dokumentu oraz ułatwia wprowadzanie zmian w strukturze dokumentu.

Do znacznego ułatwienia i przyspieszenia edytowania styli używane są preprocesory oraz frameworki. Framework dla CSS to zbiór gotowych stylów, który ułatwia i znacznie przyspiesza budowanie stron. Jednym z najbardziej popularnych jest Bootstrap. Dzięki niemu tworzenie responsywnych stron jest dużo prostsze. Zawiera także gotowe szablony dla typografii, formularzy, przycisków nawigacji i innych elementów interfejsu. Preprocesory takie jak LESS oraz SASS wprowadzają elementy znane z języków programowania czyli między innymi funkcje, instrukcje warunkowe, pętle. Oczywiście taki kod wygenerowany przez preprocesor nie mógłby zostać poprawnie wczytany i zinterpretowany przez przeglądarkę, zatem musi zostać najpierw zamieniony na wynikowy kod CSS.
