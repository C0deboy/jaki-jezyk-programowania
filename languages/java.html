Zastosowanie

Java to uniwersalny język programowania, który jest w pełni przenośny, gdyż jest interpretowany przez wieloplatformową maszynę wirtualną Javy (Java Virtual Machine). Dlatego powstał slogan „Napisz raz, pracuj wszędzie!”. Java jest używana do tworzenia oprogramowania, gier oraz backendowych systemów stron. Wykorzystywana na duża skalę w systemach bankowych. Ponadto, system operacyjny Android działa w środowisku Javy, co umożliwia także tworzenie aplikacji lub gier na Androida.

Popularne aplikacje napisane w Javie to Gmail, Minecraft, OpenOffice, Eclipse, NetBeans. Do najbardziej znanych stron napędznych przez Javę należa LinkedIn.com, Netflix.com i Amazon.com.

Poziom skomplikowania

Java została zaprojektowana tak, aby była przyjazna dla początkujących, tym samym łatwiejsza niż C++. Składnia Javy jest oczyszczoną wersją składni języka C++, zrezygnowano z jego wielu kłopotliwych i nie eleganckich właściwości. Java posiada także wiele bibliotek, z gotowymi rozwiązaniami, podczas gdy w C++ musimy większośc rzeczy napisać sami. Osoba ucząca się Javy, ma relatywnie mniejsze prawdopodobieństwo na strzelanie sobie w nogę. Java to język wysokiego poziomu, co oznacza, że Java abstrahuje (zajmuję się tym za Ciebie) od sprzętowych detalów takich jak np. zarządzanie pamięcią, więc możesz skupić się na programowaniu zamiast martwić się o rzeczy, które wielu uważa za żmudne, czy trudne. Z drugiej strony Java jest rozwlekła, to znaczy , że trzeba napisać dużo więcej kodu, niż w przypadku innych języków, aby dostać podobną, działającą aplikacje. Może być to demotywujące dla początkujących.

Wydajność

Java jest językiem statycznym, co oznacza, że twój kod zostanie sprawdzony w celu znalezienia błędów zanim zostanie przetworzony, więc łatwiej je namierzyć. Dodatkowo, statyczne języki działają szybciej niż dynamiczne, gdyż podczas działania aplikacji zasoby nie są marnowane na sprawdzanie poszczególnych definicji w twoim kodzie.

Javę cechuję także silne typowanie, to znaczy, że wyrażenia mają ustalony typ danych. Zapobiega to powstawaniu dziwnych i niespodziewanych błędów, więc kod jest łatwiejszy w utrzymaniu. Javie zarzucano, że jest dużo wolniejsza niż języki natywnie kompilowane, jak np. C++. Jednak odnosi się to do starszych wersji. Obecnie wydajność Javy została poprawiona tak, że może konkurować z tradycyjnymi kompilatorami. Głównie dzięki kompilatorowi JIT (Just in time), który potrafi optymalizować pod kątem szybkości te części kodu które są najczęściej używane, oraz mechanizmom takim jak odśmiecanie pamięci (ang. garbage collecting).

Popularność

Java jest najpopularniejszym językiem programowania według TIOBE index i to ze sporą przewagą, głównie dzięki swojej wieloplatformowości.

Posiada również drugą co do wielkości społeczność na StackOverFlow, z liczbą 1.200.000 pytań, rozwiązujący chyba każdy problem. Jest również drugim najczęściej tagowanym językiem na GitHub, gdzie można znaleźć ponad 374 000 projektów. System oceniania potrafi wyróżnić naprawdę te przydatne, które mogą posłużyć jako wzorce, inspiracja czy biblioteki, których sam będziesz używał. Gdy chodzi o spotkania na żywo, Java na Meetup.com Java posiada ponad 898,146 z ponad 1,762 spotkaniami.

Zarobki

Według wynagrodzenia.pl miesięczne wynagrodzenie całkowite (mediana) na tym stanowisku wynosi 6 500 PLN brutto. Co drugi programista Javy otrzymuje pensję od 4 536 PLN do 9 147 PLN. 25% najgorzej wynagradzanych programistów zarabia poniżej 4 536 PLN brutto. Na zarobki powyżej 9 147 PLN brutto może liczyć grupa 25% najlepiej opłacanych programistów. Natomiast według Payscale.com średnie zarobki miesięczne programisty w Anglii wynoszą 2828.5£.

Przyszłość języka

Programiści Javy mają duże zapotrzebowanie na rynku ze względu na jej popularność. Jest jednym z najbardziej opłacalnych oraz najbardziej pożądanych języków. Pracodawcy oferują atrakcyjne warunki zatrudnienia dla tych, którzy wiedzą, jak pracować z Javą. Java cały czas jest rozwijana i ulepszana, aby utrzymać się w topce na rynku. Mimo to, że często krytykowano ten język i wiele języków próbowało zastąpić Jave, wyszła z tego zwycięsko i będzie jeszcze lepsza. Bugi są dosyć szybko naprawiane a ilość narzędzi i bibliotek dostępnych dla Javy nie będzie łatwa do zastąpienia. W kolejnych latach Java raczej nigdzie się nie wybiera, będzie kontynuować swoja dominacje.
